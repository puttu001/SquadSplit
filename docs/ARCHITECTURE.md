# SquadSplit — Architecture Reference

## Overview

SquadSplit is a full-stack monorepo built with pnpm workspaces.

```
SquadSplit/
├── apps/
│   ├── backend/      Node.js + Express + TypeScript (REST API + WebSockets)
│   └── frontend/     React 18 + Vite + TypeScript (SPA)
├── packages/
│   └── shared/       Shared constants & types (imported by both apps)
└── docs/
```

---

## Tech Stack

| Layer        | Technology                                                  |
|--------------|-------------------------------------------------------------|
| Backend      | Node.js 20, Express 4, TypeScript 5                         |
| ORM          | Prisma 5 (PostgreSQL)                                       |
| Cache / Jobs | Redis                                                       |
| Real-time    | Socket.io 4                                                 |
| Validation   | Zod                                                         |
| Auth         | JWT (access + refresh tokens), bcryptjs                     |
| Email        | Nodemailer (SMTP)                                           |
| Frontend     | React 18, Vite, Tailwind CSS 3, React Router 6              |
| State        | Zustand (auth/notifications), TanStack Query (server state) |
| Forms        | React Hook Form + Zod                                       |
| HTTP client  | Axios (with auto-refresh interceptor)                       |

| Event              | Direction       | Payload                    |
|--------------------|-----------------|----------------------------|

### Data flow pattern

```
Page component
  └── useQuery(queryKey, featureApi.method)     ← TanStack Query for server state
  └── useMutation(featureApi.method, options)   ← mutations + toast feedback
  └── Zustand store                             ← auth / notification badge
  └── Socket.io events (via useSocket.ts)       ← real-time updates → queryClient.invalidate
```

## Database Schema (key relationships)

```
User ─── FriendRequest ─── User        (friend invitations)
User ─── Friendship    ─── User        (bidirectional after acceptance)
User ─── GroupMember   ─── Group       (many-to-many, role: ADMIN|MEMBER)
Group ── Expense ───────── ExpenseSplit ─── User  (who paid, who owes how much)
User ─── Settlement    ─── User        (payer → payee, per group)
User ─── Notification
User ─── Activity
User ─── RefreshToken
```

## Getting Started

```bash
# 1. Install dependencies
pnpm install

# 2. Copy and fill env files
cp apps/backend/.env.example  apps/backend/.env
cp apps/frontend/.env.example apps/frontend/.env

# 3. Start PostgreSQL and Redis (Docker recommended)
#    docker-compose up -d  (add docker-compose.yml yourself)

# 4. Run Prisma migrations
pnpm db:migrate

# 5. Seed demo data
pnpm db:seed

# 6. Start both apps
pnpm dev
# Backend: http://localhost:5000
# Frontend: http://localhost:5173
```
